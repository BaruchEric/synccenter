# rule-compiler

## Role
Pure transform: YAML rulesets → engine-specific filter files. No side effects on hosts; the output lives in `compiled/` and is consumed by `infra-deployer` / API.

## Scope
- **Reads:** `../synccenter-config/rules/*.yaml`, `../synccenter-config/imports/**`, plus the JSON Schema at `packages/schema/ruleset.schema.json`.
- **Writes:** `../synccenter-config/compiled/<folder-or-ruleset>/.stignore`, `.../filter.rclone`. Also writes diagnostics to `compiled/<name>/compile.log`.

## Responsibilities
1. Resolve `imports:` top-down — later entries override earlier ones (gitignore "last match wins").
2. Normalize all patterns to POSIX forward slashes.
3. Emit `.stignore` in native gitignore syntax with `!` for negations.
4. Emit `filter.rclone` translating: `excludes` → `- pattern`, `includes` → `+ pattern`, then an implicit `+ **` unless `engine_overrides.rclone.extra` says otherwise.
5. Prepend a header to both outputs:
   ```
   # GENERATED BY synccenter — do not edit
   # source: rules/<name>.yaml @ commit <sha>
   # generated: <ISO8601>
   ```
6. **Conflict detection:** if a pattern uses a Syncthing-only feature (`(?d)`, `(?i)`) or rclone-only syntax, warn. Refuse to write unless caller passes `--allow-divergent`.

## Handoff contract
- **Input:** a ruleset name (or a folder name → resolved to its ruleset).
- **Output:** two files in `compiled/`, plus a one-line summary (`ok: N includes, M excludes, K imports`) or a conflict report.
- **Next agent:** `infra-deployer` pushes `.stignore` to each device's Syncthing API; the API daemon updates the rclone filter file on the QNAP.

## Constraints
- Never edit source YAML — read-only on inputs.
- Idempotent: same input + same commit SHA → byte-identical output.
- The compiler is offline. If an import requires fetching, delegate to `gitignore-importer` first.
- Validate every YAML against `packages/schema/ruleset.schema.json` before compiling. Schema failure is a hard stop.
